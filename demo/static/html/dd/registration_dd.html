Introduction

Purpose

Nature v1.0 is an inventory management system for the dairy farming market segment. The Registration feature will consist of a user credential protected UI that interacts with a service that manages user registration.

Scope

Two tools will be developed. A CLI tool for rapid creation of new objects for load testing and a UI tool for CRUD management of all objects. Back-end Perspective Each client will be represented by a Customer object. All users and assets will be mapped to this Customer object to ensure multi-tenancy security. Customer registration will require name, address, contact, telephone, and email_address. Each Customer will be mapped to 1:N User objects. User registration will require first_name, last_name, employer, position, email_address as username and a password for security. Each user of the CLI tool or UI must have an User object that provides authorization via login credentials: email address and password, and notification alerting via email address. Each User is assigned read-only or read-write access to 1:N asset objects. Front-end Perspective User login trades email address/password for authentication token and authorization ACL Partner User login loads a landing page listing all Initiatives with attributes hidden. Every Initiative icon can be expanded to display all attributes. Many Initiative attributes can be edited. Ampush User login loads a landing page listing all mapped Partners with attributes hidden. Every Partner icon can be expanded to display all attributes including all mapped publishers and all mapped Initiatives. Many Partner attributes can be edited. A click of any Partner loads a new landing page listing all mapped Initiatives with attributes hidden. Every Initiative icon can be expanded to display all attributes. Many Initiative attributes can be edited. *Historic Initiatives may be retrieved by additional query but objects and attributes are immutable.
