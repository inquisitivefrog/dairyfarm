Introduction

Purpose

Nature v1.0 is an inventory management system for the dairy farming market segment. The Assets feature will consist of a user credential protected UI that interacts with a service that manages user registration, and management of dairy farm assets.

Scope

Two tools will be developed. A CLI tool for rapid creation of new objects for load testing and a UI tool for CRUD management of all objects. Back-end Perspective Each client will be represented by a Customer object. All users and assets will be mapped to this Customer object to ensure multi-tenancy security. User registration will require email address as username, password, first_name, last_name, employer, position. Each organization will be represented by an Agency object. Currently, the only agency supported is Ampush. Each user of the CLI or UI tool must have an User object that provides authorization via login credentials: username and password, and alerting via email address or SMS text message. Each User is assigned read-only or read-write access to 1:N Partner objects based upon Jason Web Token (JWT) and agency. Partner and Initiative objects will be accessible only by JWT obtained by user login. ie. username + password = JWT. Each User object may be modified or mapped to 1:N Partner objects Each Partner object may be modified or mapped to 1:N Initiative objects and 1:4 publishers defined by name, account_id, account_name and access_token Front-end Perspective User login trades username/password for authentication token and authorization ACL Partner User login loads a landing page listing all Initiatives with attributes hidden. Every Initiative icon can be expanded to display all attributes. Many Initiative attributes can be edited. Ampush User login loads a landing page listing all mapped Partners with attributes hidden. Every Partner icon can be expanded to display all attributes including all mapped publishers and all mapped Initiatives. Many Partner attributes can be edited. A click of any Partner loads a new landing page listing all mapped Initiatives with attributes hidden. Every Initiative icon can be expanded to display all attributes. Many Initiative attributes can be edited. *Historic Initiatives may be retrieved by additional query but objects and attributes are immutable.
